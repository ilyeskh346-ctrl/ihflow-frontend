أريد إضافة نظام ON/OFF للبوتات داخل قسم Bots في الـ SaaS الحالي.

⚠️ مهم جدًا:

* المطلوب فقط تعديلات تخص:

  * Replit frontend
  * Supabase database
* لا تعدّل أي شيء في n8n
* لا تغيّر أي webhook أو automation
* لا تكسر التصميم الحالي
* لا تغيّر architecture المشروع

# 🎯 الهدف

كل Bot يجب أن يملك حالة:

* ON → البوت مفعل
* OFF → البوت معطل

ويتم حفظ الحالة داخل Supabase وربطها بالواجهة فقط.

---

# 🔥 المطلوب تنفيذه

## 1. إضافة field داخل جدول bots في Supabase

إذا لم يكن موجودًا:

```sql id="4dazj0"
is_active BOOLEAN DEFAULT true
```

⚠️ جميع البوتات القديمة يجب أن تبقى:

```text id="qq93x5"
is_active = true
```

---

## 2. تحديث frontend لجلب is_active

عند تحميل البوتات:

* اجلب قيمة is_active
* أضفها إلى bot object

---

## 3. إضافة Toggle ON/OFF داخل Bot Card

داخل كل Bot Card أضف Toggle Switch حديث ومتناسق مع التصميم الحالي:

* ON → أخضر
* OFF → رمادي أو أحمر خفيف

---

## 4. عند تغيير الـ Toggle

قم بتحديث Supabase مباشرة:

### ON:

```js id="zw6vjq"
is_active = true
```

### OFF:

```js id="jjlwm1"
is_active = false
```

⚠️ استخدم optimistic UI إن أمكن.

---

## 5. تحديث واجهة البوت عند OFF

إذا:

```js id="6m52r9"
bot.is_active === false
```

قم بـ:

* إظهار badge أو label:
  "Bot Disabled"
* تقليل opacity للكارت قليلًا
* تعطيل أزرار التشغيل أو الإرسال إن وجدت
* منع أي UI action خاص بالبوت

⚠️ لا تحذف البوت أو بياناته.

---

## 6. تحسين UX

أضف:

* loading state أثناء التحديث
* success feedback
* error handling إذا فشل تحديث Supabase
* rollback للحالة إذا فشل الحفظ

---

## 7. تنظيف الكود

* لا تكرر logic التبديل
* أنشئ function موحدة:

```js id="w08tna"
toggleBotStatus(botId, isActive)
```
